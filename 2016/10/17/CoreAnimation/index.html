Oct 17, 2016 | Hits

CAAnimation 类是animation的基类，实现了NSCoding、NSCopying等协议。本文重点关注的是CAMediaTiming、CAAction这两个协议。CAAnimation同时也是Core Animation所有动画类型的抽象基类。

CAMediaTiming协议

CAMediaTiming 协议由layers和animation实现，它是一个分层定时系统的模型。可以调整时间,包括持续时间,速度,重复次数。

  • beginTime : 设置动画的开始时间，用来延迟动画的开始执行时间，比如说，想延迟2s才开始执行，设置为CACurrentMediaTime() + 2, CACurrentMediaTime()表示当前图层的时间;

  • duration : 设置动画持续时间;

  • speed : 设置动画运行速度，speed是一个速度倍增器。默认该属性的值为1.0，若该值设为2.0将会引起动画运行速度为原来速度的两倍。该属性的图层会影响到图层的子图层，如果子图层和父图层都拥有2.0的速度，则子图层的速度将会是原来的4倍。

  • timeOffset : 让动画一开始就出现在原本是在某个时刻才会出现的状态，使用在暂停动画和开始动画。（我的理解是设置穿越时间)

  • repeatCount : 动画重复次数，无限循环可以设置为HUGE_VALF或者MAXFLOAT

  • repeatDuration : 动画重复的时间。

  • autoreverses : BOOL 类型，设置为YES将引起动画在指定时间内执行相反的动画，与repeatcount属性配合使用，可来回反复执行动画。

  • fillMode : 设置当前对象在非active时间的行为。

    kCAFillModeRemoved: fillMode默认值，动画开始前喝动画结束后，动画对layer没有影响，动画结束后，layer会恢复到之前的状态。

    kCAFillModeBackwards: 该fillMode模式下，在动画开始前，图层将显示动画的开始值，即使在图层树中的图层对象包含了一个不同的值。

    kCAFillModeForwards: 该fillMode模式下，在动画结束后，图层会一直保持着动画最后的状态。

    kCAFillModeBoth: 该fillMode模式下，就是kCAFillModeBackwards和kCAFillModeForwards的组合。

Core Animation使用缓冲来使动画移动更平滑更自然，而不是看起来的那种非常机械的效果，CAMediaTimingFunction类给我们提供了对动画和自定义缓冲曲线的处理方法以及属性。

    kCAMediaTimingFunctionLinear : 创建一个线性的计时函数，CAAnimation的timingFunction属性的默认函数，适用于那些立即加速并且保持匀速到达终点的场景。

    kCAMediaTimingFunctionEaseIn : 创建了一个慢慢加速然后突然停止的过程，适用于自由落体的场景。

    kCAMediaTimingFunctionDefault : 和kCAMediaTimingFunctionEaseInEaseOut的效果非常类似，就是加速和减速过程会慢一点。

CAPropertyAnimation类

CAPropertyAnimation类是CoreAnimation框架中，很重要的类，继承了CAAnimation，它有两个非常常用的子类CABaseAnimation和CAKeyFrameAnimation。

  • CAPropertyAnimation的属性:
    keyPath : 返回创建CAPropertyAnimation时指定的动画参数；

    additive : 指定该属性动画是否以当前动画效果为基础(默认是NO)；

    cumulative : 指定动画是否为累加效果(默认是NO)；

    valueFunction : 是一个CAValueFunction对象，该属性负责对属性改变的插值计算，系统已经提供了默认的插值计算方式；

  • CAPropertyAnimation的方法:

CABasicAnimation类:

动画其实就是一段时间内发生的改变，最简单的形式就是从一个值改变到另一个值，这也是CABasicAnimation最主要的功能。CABasicAnimation是CAPropertyAnimation的一个子类。

  • CABasicAnimation的属性:
    fromValue : 动画开始之前属性的值;

    toValue : 动画结束之后的值;

    byValue : 动画执行过程中改变的值;

      通过组合这三个属性就可以有很多种方式来指定一个动画的过程。它们被定义成id类型而不是一些具体的类型是因为属性动画可以用作很多不同种的属性类型，包括数字类型，矢量，变换矩阵，甚至是颜色或者图片。

      fromValue，toValue和byValue属性可以用很多种方式来组合，但为了防止冲突，不能一次性同时指定这三个值。例如，如果指定了fromValue等于2，toValue等于4，byValue等于3，那么Core Animation就不知道结果到底是4（toValue）还是5（fromValue + byValue）了。他们的用法在CABasicAnimation头文件中已经描述的很清楚了，所以在这里就不重复了。总的说来，就是只需要指定toValue或者byValue，剩下的值都可以通过上下文自动计算出来。

      id类型可以包含任意由NSObject派生的对象，但是有时候希望对一些不直接从NSObject继承的属性类型做动画，这就需要把这些值用一个对象来封装或者转成一个对象

CAKeyframeAnimation类:

CAKeyframeAnimation同样是CAPropertyAnimation的一个子类，它依然作用于单一的一个属性，但是和CABasicAnimation不一样的是，它不限制于设置一个起始和结束的值，而是可以根据一连串随意的值来做动画。

  • CAKeyframeAnimation的属性:
    values : 存放一连串关键帧的数组，动画对象会在指定时间（duration）内，依次现实values数组中的每一个关键帧。

    path : 可以存放一个数组，比如设置一个CGPathRef\CGMutablePathRef，让层跟着路径移动，path只对CALayer的anchorPoint和position起作用。如果设置了path，那么values将被忽略。

    keyTimes : 可以为对应的关键帧指定对应的时间点，其取值范围为0到1.0，keyTimes中的每一个时间值都对应values中的每一帧，当keyTimes没有设置的时候，各关键帧的时间是平分的，CABasicAnimation可以看作是只有2个关键帧的CAKeyframeAnimation。

    timingFunctions : 存放着CAMediaTimingFunction对象的数组。

    calculationMode : 计算模式，其主要针对的是每一帧的内容为一个坐标点的情况，也就是对anchorPoint和position进行的动画。当在平面坐标系中有多个离散的点的时候，可以是离散的，也可以是直线相连后进行插值计算，也可以使用圆滑的曲线将他们相连后进行插值计算。
    提供的几种calculationMode:
    kCAAnimationLinear : 默认值，表示当关键帧为坐标点的时候，关键帧之间直接直线相连进行插值计算。
    kCAAnimationDiscrete : 离散的，就是不进行插值计算，所有关键帧直接逐个进行显示。
    kCAAnimationPaced : 使得动画均匀进行，而不是按keyTimes设置的或者关键帧平分时间，此时keyTimes和timingFunctions无效。
    kCAAnimationCubic : 对关键帧为坐标点的关键帧进行圆滑曲线相连后插值计算，这里主要目的是使得运行的轨迹变得圆滑。
    kCAAnimationCubicPaced : 在KCAAnimationCubic的基础上使得动画运行变得均匀，就是系统时间内运动的距离相同，此时keyTimes以及timingFunction也是无效的。

    tensionValues : 控制曲线的“紧密度”。正值更紧，负值更圆。

    continuityValues : 控制如何连接。正值尖锐角，负值反转角。

    biasValues : 控制曲线发生的位置。正值在移动曲线之前控制点，负值在控制点之后移动。

    rotationMode : 是否沿路径动画对象或旋转以匹配(默认为nil);
    kCAAnimationRotateAuto : 路径切线，自动。
    kCAAnimationRotateAutoReverse : 路径切线，自动翻转。

CASpringAnimation类:

CASpringAnimation是继承自CABasicAnimation,是iOS 9 才引入点动画类，用于制作弹簧动画。

  • CASpringAnimation的属性:
    mass : 质量，影响图层运动时的弹簧惯性，质量越大，弹簧拉伸和压缩的幅度越大，默认是1 必须大于0。

    stiffness : 刚度系数（劲度系数／弹性系数），刚度系数越大，形变产生的力就越大，运动越快，必须大于0 默认是100。

    damping : 阻尼系数，防止弹簧伸缩的系数，阻尼系数越大，停止越快，默认是10 必须大于或者等于0。

    initialVelocity : 初始速率，动画视图的初始速度大小，速率为正数时，速度方向与运动方向一致，速率为负数时，速度方向与运动方向相反，默认是0。

    settlingDuration : 结算时间，返回弹簧动画到停止时的估算时间，根据当前的动画参数估算，通常弹簧动画的时间使用结算时间比较准确

CATransition类:

CATransition是CAAnimation的子类，用于做转场动画，能够为层提供移出屏幕和移入屏幕的动画效果。UINavigationController就是通过CATransition实现了将控制器的视图推入屏幕的动画效果。

  • CATransition的属性:
    type : 动画过渡类型。

    subtype : 动画过渡方向。

    startProgress : 动画起点，在整个动画的百分比。

    endProgress : 动画的终点，在整个动画的百分比。

    filter : 转场动画过滤器。

  • 普通的转场动画效果:
    kCATransitionFade : 逐渐消失。
    kCATransitionMoveIn : 移入。
    kCATransitionPush : 平移。
    kCATransitionReveal : 显露。
    kCATransitionFromRight : 往右。
    kCATransitionFromLeft : 往左。
    kCATransitionFromTop : 向上。
    kCATransitionFromBottom : 向下。
