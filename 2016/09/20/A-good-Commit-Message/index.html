如何写好一个优雅的提交信息？

Sep 20, 2016 | Hits

  一个优雅的提交信息，不仅可以节省时间，还可以降低沟通成本，让我们的协作更加的高效、简洁。

一个好的提交信息至少有三个重要的目的：

  • 加速审查的过程。
  • 帮助我们写一个好的发布日志。
  • 五年以后，帮助未来的维护者（也许是你自己！），找到这里为什么更改了特定的代码或者为什么增加了特定的功能。

你提交信息的结构应该像这样：

From: http://git-scm.com/book/ch5-2.html

简短（50个字符或者更少）的摘要
如果有必要的话，写一个更详细的说明。大概有72个字符。一般情况下，第一行是电子邮件信息和其他文本信息。用空行分离主体信息和摘要是非常必要的（除非忽略主体信息）。工具的rebase会混淆两个的合并。

空一行后编写要点，罗列出改动原因、主要变动、以及需要注意的问题。如果有必要，在最后提供对应的网址。

- 要点在这一行

- 通常根据习惯使用“-”或者“*”来作为要点的开始，要点之间用空行隔开

这样做：

  • 在总结里面必须描述你做了什么东西，即使是你让别人做的。这行的开始用“Fix”、“Add”，“Change” 取代“Fixed”、“Added”、“Changed”。
  • 要始终保持第二行是空白。
  • 换行提交信息（使提交信息可读，而无需在gitk水平滚动）。

不要这样做：

  • 不要用句号结束总结行-它是一个标题，标题是没有句号结尾的。

小贴士

  • 如果很难总结你的提交，可能是它包含了多个逻辑变更或者错误修复，更好的方式是使用 git add -p 分成多次提交。
